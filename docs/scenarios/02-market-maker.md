# 演示脚本 02: 做市商双边报价 (流动性与基差联动)

## 场景目标
演示专业做市商如何利用系统自动化工具：
1. **基差联动双边报价**：一键以期货价为基准同步发布买/卖单。
2. **市场价差收窄**：展示做市商加入后，市场流动性的显著提升。
3. **风控额度监控**：演示双边挂单对机构保证金占用的累加效应。

## 角色分配
*   **角色 A**: 宏源做市 (做市商) - 信用评级: **AA+**
*   **其他角色**: 系统内已有的挂机报价 (或买/卖方角色)

## 演示准备
1. 建议先执行场景 01，使市场存在一定的买卖价差（如买 3830 / 卖 3850）。
2. 窗口登录 `宏源做市` 身份。

## 演示流程

### 步骤 1: 做市商介入并激活“基差套利”模式
1.  **操作**: 做市商选择 `螺纹钢`，开启 `基差报价 (PRO)`。
2.  **动作**: 
    *   此时期货报价为 `3820`。做市商为了提供流动性，设置升贴水为 `+25`。
    *   点击 `发布双边快速报价`。
3.  **结果**: 
    *   系统瞬间生成两笔报价：买单 `3840` (3820+25-5) 和 卖单 `3850` (3820+25+5) —— *假设做市商内置价差为 ±5*。
    *   观察行情版块：`买1` 立即更新为 `3840`，显著优于场景一中的 `3830`。
4.  **解说**: *“做市商通过‘基差快速联动’工具，一键报出了具有竞争力的双边价格。这不仅展示了系统对复杂指令的高速处理能力，更体现了清算所在极速行情下自动化清算和风控的支撑能力。”*

### 步骤 2: 验证保证金占用
1.  **操作**: 做市商查看报价面板底部的 `结算风控预估`。
2.  **视觉重点**: 展示即便是在复杂的双边报价下，系统也已预先计算好了基于净头寸或双边总量的保证金预留。
3.  **资产核实**: 进入“我的资产”，核实两笔挂单均已处于 `待成交` 状态。

## 测试验证点
- [ ] 基差双边报价：是否能基于期货价一次性生成两笔（买/卖）挂单？
- [ ] 价格优先：做市商报出的更优价是否排在五档行情的最顶端？
- [ ] 风险看板：报价单底部的“保证金预估”是否反映了交易的资金影响？
- [ ] 信用标识：做市商身份旁的 AA+ 评级是否正确显示？
