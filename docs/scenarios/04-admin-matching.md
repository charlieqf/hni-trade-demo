# 演示脚本 04: 管理员人工撮合 (异常处理与监管审计)

## 场景目标
演示清算所作为第三方公信平台，在复杂或争议业务下的干预能力：
1. **全局指令视角**：管理员如何调取并审核全场非相交订单。
2. **强制合规匹配**：演示在特殊业务背景（如协议平仓）下的人工干预撮合。
3. **审计留痕验证**：展示系统如何区分标记人工操作，确保清算链路的可追溯性。

## 角色分配
*   **角色 A**: 买/卖方各自挂出单据。
*   **角色 B**: HNI 监管端 (管理员) - 信用评级: **EXEMPT (监管豁免)**

## 演示准备
1. 窗口 A (买家) 报入买单：`3820`。
2. 窗口 B (卖家) 报入卖单：`3840`。
3. *此时买卖价格不相交，无法自动撮合。*

## 演示流程

### 步骤 1: 管理员登录撮合工作台
1.  **操作**: 登录 `HNI 监管端`。点击顶部的 `人工撮合` 标签。
2.  **动作**: 
    *   在工作台中，左侧显示全场买单（含海汽大宗的 3820），右侧显示全场卖单（含沙钢贸易的 3840）。
    *   管理员同时勾选这两笔订单。
    *   **亮点验证**: 系统已实现跨窗口实时同步，管理员工作台会即时显现其他交易员报入的订单，无需刷新。
3.  **智能辅助**: 观察系统底部自动弹出的撮合建议，展示两笔订单的明细对比及信用状况。

### 步骤 2: 执行强制人工撮合
1.  **动作**: 管理员在底部输入撮合价格（如协议价 `3830`），填入成交数量，并输入审计备注 “协议平仓”。点击 `确认执行人工干预撮合`。
2.  **视觉重点**: 
    *   系统反馈撮合成功。
    *   观察行情版块，最新的成交价即刻更新为 `3830`。
3.  **解说**: *“管理员拥有跨越报价壁垒的最高清算权限，能够针对线下达成的特定交易意向，在系统中进行有据可查的手动撮合。”*

### 步骤 3: 审计与结算核实
1.  **操作**: 管理员进入 `我的资产` -> `结算历史`。
2.  **关键细节**: 
    *   由于是人工通过工作台处理，该笔成交记录的“撮合方式”被醒目地标记为亮黄色的 **人工撮合 (MANUAL)** 标签。
    *   查看仓单，同样记录了管理员干预后的最终成交参数及审计备注。
3.  **解说**: *“清算平台对所有业务动作均有完整的审计路径。正如各位所见，系统能通过自动化标记明确区分交易是来源于引擎逻辑还是专业监管的人工介入。”*

## 测试验证点
- [x] 管理员是否能全局透视非本机构的所有挂单？
- [x] 人工撮合流程：管理员是否可以自定义成交价格、数量及备注？
- [x] 存证审计：结算历史中是否显示了亮黄色的“人工撮合”标签？
- [x] 权限隔离：普通买卖家是否**无法**进入“人工撮合”工作台？
